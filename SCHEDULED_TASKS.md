# 定时任务配置说明

本文档说明了为 `code-consistency-agent-skill` 项目配置的自动化定时任务。

## 任务概述

为了维护 GitHub 贡献图（绿墙）并保持项目活跃度，已配置两个定时任务，在每个工作日自动执行代码提交。

## 定时任务详情

### 1. 上午任务 (github_green_wall_morning)

- **执行时间**: 每个工作日（周一至周五）上午 10:00（北京时间）
- **Cron 表达式**: `0 0 10 * * 1-5`
- **执行内容**: 运行 `auto-commit.sh` 脚本，更新项目活动日志并提交到 GitHub

### 2. 下午任务 (github_green_wall_afternoon)

- **执行时间**: 每个工作日（周一至周五）下午 16:00（北京时间）
- **Cron 表达式**: `0 0 16 * * 1-5`
- **执行内容**: 运行 `auto-commit.sh` 脚本，更新项目活动日志并提交到 GitHub

## 自动提交脚本说明

脚本位置: `auto-commit.sh`

### 脚本功能

1. 自动更新 `ACTIVITY.md` 活动日志文件
2. 记录每次执行的时间戳
3. 使用随机的提交消息类型，模拟真实的开发活动
4. 自动推送到 GitHub 远程仓库

### 提交消息类型

脚本会随机选择以下提交消息之一：

- `docs: 更新项目文档`
- `docs: 完善使用说明`
- `docs: 优化代码注释`
- `chore: 维护项目配置`
- `docs: 添加使用示例`
- `docs: 更新活动日志`

## 时间安排说明

- **工作日**: 周一至周五
- **休息日**: 周六、周日（不执行任务）
- **时区**: 北京时间（GMT+8）
- **频率**: 每个工作日 2 次提交

## 预期效果

### GitHub 贡献图

每个工作日会产生 2 次提交记录，在 GitHub 贡献图上显示为绿色方块，全年预期贡献次数：

- 每周: 10 次提交（5 个工作日 × 2 次/天）
- 每月: 约 40-44 次提交
- 每年: 约 520 次提交

### 项目活跃度

- 保持项目的持续更新状态
- 展示项目的活跃维护
- 记录完整的项目活动历史

## 手动执行

如果需要手动触发提交，可以直接运行：

```bash
cd /home/ubuntu/code-consistency-agent-skill
bash auto-commit.sh
```

## 注意事项

1. **GitHub 权限**: 确保 GitHub CLI (`gh`) 已正确配置并具有推送权限
2. **网络连接**: 定时任务执行时需要网络连接以推送到 GitHub
3. **分支名称**: 脚本会尝试推送到 `main` 或 `master` 分支
4. **活动日志**: `ACTIVITY.md` 文件会持续增长，建议定期归档

## 任务管理

定时任务由 Manus 平台管理，会在指定时间自动触发执行。任务状态和历史记录可以在 Manus 控制台查看。
